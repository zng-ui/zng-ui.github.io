P Widget interactivity changed.

Note that there are multiple specific events for interactivity changes, on_enable, on_disable, on_block and on_unblock are some of then.

Note that an event is received when the widget first initializes in the widget info tree, this is because the interactivity changed from None, this initial event can be detected using the is_new method in the args.

You can preview this event using on_pre_interactivity_changed. Otherwise the handler is only called after the widget content has a chance to stop propagation.
